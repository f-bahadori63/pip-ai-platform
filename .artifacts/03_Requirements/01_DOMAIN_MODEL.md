# Purpose

Define the main business domains and their relationships in PIP.

---

# Core Domain

## Project Management Domain

Responsible for managing the complete lifecycle of industrial projects.

---

# Business Domains

## 1. Project Domain

Main entities:

- Project

- Contract

- Milestone

- Phase

Purpose:

Manage project structure and lifecycle.

---

## 2. Planning Domain

Main entities:

- Schedule

- Activity

- Baseline

- Progress Update

Purpose:

Control project timeline and performance.

---

## 3. Cost Domain

Main entities:

- Budget

- Cost Item

- Actual Cost

- Forecast

Purpose:

Manage project financial performance.

---

## 4. Risk Domain

Main entities:

- Risk

- Issue

- Mitigation Action

Purpose:

Identify and control project uncertainties.

---

## 5. Document Domain

Main entities:

- Document

- Revision

- Approval

Purpose:

Manage project information.

---

## 6. User Domain

Main entities:

- User

- Role

- Permission

Purpose:

Control access and responsibilities.

---

## 7. AI Intelligence Domain

Main entities:

- Recommendation

- Prediction

- Knowledge Item

Purpose:

Provide intelligent analysis and decision support.

---

# Domain Relationship

Project is the central entity.

Other domains provide information and services around the project lifecycle.
