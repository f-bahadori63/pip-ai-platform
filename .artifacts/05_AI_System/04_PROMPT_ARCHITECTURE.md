# Purpose

Define the structure and management approach for AI prompts in PIP.

---

# Prompt Structure

Each prompt consists of:

## 1. Role Definition

Defines AI responsibility.

Example:

"You are a project control assistant."

---

## 2. Context

Provides required information.

Examples:

- Project data
- Documents
- User request

---

## 3. Task Definition

Defines the required action.

Examples:

- Analyze
- Compare
- Predict
- Recommend

---



## 4. Output Format

Defines response structure.

Examples:

- Table
- Report
- Summary
- Recommendation

---



# Prompt Categories



## Project Management Prompts

Used for:

- Project analysis
- Reporting
- Decision support

---



## Planning Prompts

Used for:

- Schedule review
- Delay analysis
- Progress evaluation

---



## Risk Prompts

Used for:

- Risk identification
- Risk assessment
- Response planning

---



## Document Prompts

Used for:

- Document analysis
- Knowledge extraction

---



# Prompt Management Principles

- Prompts must be version controlled
- Prompts must be reusable
- Prompts must be tested
- Prompts must have clear objectives
